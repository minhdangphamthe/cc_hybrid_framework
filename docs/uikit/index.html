Mid-core / Casual / Hypercasual UI Catalog

A reusable UI component catalog with built-in behavior guidance (visual feedback + VFX/animation + performance techniques). Media in this playbook is generated (PNG + GIF) so you can ship and maintain it without licensing issues.

How to use
Use the left sidebar to jump to a category. Use Search to filter both the navigation and the cards. Each component card includes: Use, Visual feedback, and Tech notes (cache/preload/pooling/virtualization).

What this solves

  • Reusable popups, lists, scrolling panels, economy UI, and shop/leaderboard compositions.
  • Consistent motion language (press, enter/exit, micro-feedback).
  • Fast rendering: pooling, virtualization, diff update, asset preloading.

Recommended baseline behaviors

  • All overlays must be queued and input-locked.
  • All long lists must be virtualized (windowing + pooled cells).
  • All counters should be event-driven, not polled.

Performance checklist

Cache & preload

  • Warm up: fonts, atlases, UI sprites, common prefabs.
  • Bundle split: core, meta, gameplay, shop.
  • Preload per feature entry (e.g. shop tab images only when first opened).

Pooling

  • Pool: toast, floating text, confetti emitters, list cells, item cards.
  • On despawn: stop tweens/timers, remove listeners, reset state.
  • Cap concurrent FX (coins/confetti) to avoid spikes.

Scrolling fast

  • Virtualize: render only visible range + small buffer.
  • Diff update: update only changed cells (by id), not rebuild list.
  • Throttle: scroll callbacks, text layout, and number counters.

Animation budget

  • Prefer transform + opacity (avoid layout reflow during tweens).
  • Batch micro animations (icon pop, badge ping) with shared timeline.
  • Disable offscreen animations; pause when hidden.
Maintaining this playbook
Add a new component by creating a new card (HTML) and dropping its PNG/GIF into assets/img and assets/gif. Keep all shared styling in playbook.css and all behaviors in playbook.js.
